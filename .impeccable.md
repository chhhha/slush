## Design Context

### Users
- 회사 직원들이 슬러시 기계 상태를 확인하는 내부 현황판
- 모바일로 빠르게 확인하는 경우와 PC에서 여유롭게 확인하는 경우가 반반
- 핵심 작업: "지금 슬러시 먹을 수 있나?" 를 1-2초 안에 파악
- 관리자는 PC에서 기계 상태 관리, 공지 작성, 로그 확인

### Brand Personality
- **친근하고 재미있는** (Friendly, Playful, Approachable)
- 회사 복지 이벤트답게 밝고 즐거운 분위기
- 딱딱한 업무 도구가 아닌, 직원들이 웃으며 접하는 서비스
- 슬러시(빙과류) 컨셉에 맞는 시원하고 상쾌한 감성도 어울림

### Aesthetic Direction
- **톤**: 밝고 장난기 있되, 정보 전달은 명확하게
- **색상**: 상태별 5색 체계 유지 (노란/파란/초록/회색/빨간), 전체 톤은 시원한 블루 계열 베이스 권장
- **타이포**: Geist Sans (현재 사용 중) — 깔끔하면서 현대적
- **아이콘**: Lucide React — 일관된 선 아이콘 사용
- **테마**: 라이트/다크 모두 지원 (system default)
- **레퍼런스**: 특정 레퍼런스 없음, 자유로운 방향
- **안티 레퍼런스**: 기업 대시보드 느낌 (차갑고 딱딱한), 과도한 데이터 시각화

### Design Principles
1. **한눈에 파악**: 상태를 색상과 아이콘으로 즉시 인지할 수 있어야 한다. 읽는 게 아니라 보는 것.
2. **즐거운 경험**: 슬러시 이벤트의 즐거움이 UI에서도 느껴져야 한다. 마이크로인터랙션, 부드러운 전환 활용.
3. **모바일 우선**: 직원 대부분이 모바일로 빠르게 확인. 터치 타겟 충분히, 핵심 정보 최상단.
4. **시원한 감성**: 슬러시/빙과류 컨셉에 맞는 시원하고 상쾌한 비주얼 톤 유지.
5. **단순하게**: 불필요한 장식 배제. 상태 확인이라는 핵심 기능에 집중.

### Tech Stack
- Next.js 16 (App Router) + React 19
- Tailwind CSS v4 + shadcn/ui
- Supabase (DB + Realtime)
- next-themes (다크모드)
- Lucide React (아이콘)
- Geist Sans / Geist Mono 폰트
- tw-animate-css

### Accessibility
- 기본 수준: 적절한 색상 대비, 키보드 접근성
- 상태 표시 시 색상만이 아닌 아이콘+텍스트 레이블 병행 (현재 적용됨)
