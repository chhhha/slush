회사에서 직원들에게 슬러시 이벤트를 진행합니다.
슬러시 현황판 웹 페이지를 생성하려고 합니다.
(* 슬러시: 주스나 음료를 살짝 얼려 떠먹거나 마실 수 있게 만든 빙과류)

이 웹페이지는 직원들이 접속하여 슬러시 기계의 현재 상태를 볼 수 있을 뿐만 아니라
관리자가 기계의 현재 상태를 수동으로 관리할 수도 있습니다.

슬러시 기계를 설명하면 아래와 같습니다.
- 회사 건물의 2층, 3층, 4층 각각 슬러시 기계가 하나씩 존재 -> 2층/3층/4층 이라고 표현해야 함
- 하나의 슬러시 기계에는 통이 좌/우 존재하여 두 종류의 슬러시 저장 가능 -> 좌/우 라고 표현해야 함
따라서, 총 슬러시 통의 개수는 6개이며, 슬러시를 독립적으로 제조하기 때문에 층마다, 통마다 분리된 UI가 표시되어야 합니다.

직원이 웹페이지에 접속하여 확인할 수 있는 상태는 아래와 같습니다. 통마다 각각 확인 가능해야 합니다.
- 현재 제공 중인 슬러시 맛 정보
- 마지막 세척 후 지난 시간
- 현재 슬러쉬 상태: 준비중(노란색) / 냉각중(파란색) / 이용가능(초록색) / 품절(회색) / 고장(빨간색)
준비중 : 관리자가 슬러시가 품절임을 인지하여 준비하기 전까지의 상태
냉각중 : 슬러시가 준비 중인 상태 (이용 가능까지 남은 시간이 시간 및 분 단위로 표시됨), 시간이 경과하면 자동으로 '이용가능' 상태로 변경됨
이용가능 : 슬러시를 먹을 수 있는 상태 (이 상태가 되고 경과 시간이 시간 및 분 단위로 표시됨)
품절 : 직원이 슬러시가 비어있음을 통보한 상태
고장 : 기계가 고장나서 수리가 필요한 상태

직원이 상호작용할 수 있는 기능은 아래와 같습니다. 통마다 각각 설정 가능해야 합니다.
-  '품절' 신고 (팝업으로 재확인 필요) ('이용가능' 상태에서만 활성화됨)

관리자 기능은 아래와 같습니다. 통마다 각각 설정 가능해야 합니다.
- 관리자 페이지는 간단한 공용 비밀번호(PIN 번호)를 입력해야 접근 가능하도록 설정 (.env.local)
- 관리자가 두 명 이상이기 때문에 관리자 페이지 접속 즉시 이름을 반드시 설정받도록 해야 합니다.
- 전체 통 상태를 한눈에 볼 수 있는 '대시보드'와 개별 수정용 '편집 모드'를 분리할 것
- 슬러시 맛 설정 (텍스트)
- 슬러쉬 통 상태 변경

냉각중 : 남은 시간을 분 단위로 설정할 수 있습니다.

- 세척 완료 설정 (세척 후 지난 시간이 초기화 됨)
- 공지 팝업 내용 작성 가능해야합니다. 페이지를 열어놓았을 때, 별도의 새로고침 없이 팝업이 바로 발생해야 합니다. (팝업에는 다시 보지 않기 필수)
- PC에서 관리자 페이지를 켜고 있는 경우, 직원이 '품절'을 설정한 경우 알림을 받을 수 있어야 합니다. (ex, 시작 표시줄 알림) (설정 On/Off 가능)
- 직원이 '품절'을 설정한 경우 지정된 이메일로 메일을 전송합니다. 각 통마다 둘 이상의 이메일 주소를 등록할 수 있습니다. 이메일 전송은 Resend 라이브러리를 사용해서 구현할 것. 관리자 페이지에서 수신 이메일을 추가/삭제할 수 있는 UI를 제공할 것

기타 기능은 아래와 같습니다.
- 상태 변경에 대한 히스토리(로그)를 저장합니다. 관리자만 볼 수 있습니다. 관리자 이름 및 직원의 고유 ID(어뷰징을 막기 위한 데이터) 를 포함할 것
- 다크모드 지원 필요합니다.
- 모바일 및 PC 버전 모두 지원해야 합니다. (그에 맞는 UI)
- 악의적 사용자(직원)가 '슬러시가 품절되지도 않았는데 '품절'을 반복적으로 설정하는 것을 막을 수 있어야 합니다. (IP 뿐만 아니라 브라우저 쿠키나 로컬 스토리지를 활용한 고유 기기 ID(UUID)를 발급하여 기기 단위 차단을 병행할 것)
- 페이지를 열어놓았을 때, 상태가 변경되면 별도의 새로고침 없이 화면이 반영되어야 합니다. 이를 위해 "Supabase Realtime 기능을 반드시 사용하여 실시간 구독(Subscription)을 구현할 것
- 시각적 피드백이 중요합니다.
- 시간(냉각 남은 시간, 이용 가능 경과 시간)을 계산할 때 "모든 시간 데이터는 UTC 기준으로 DB에 저장하고, 프론트엔드에서 한국 시간(KST)으로 변환하여 렌더링할 것
- 데이터베이스 초기화 시 3개 층 x 2개 통(좌/우) 총 6개의 고정된 레코드가 존재하도록 Seed 데이터 코드를 작성할 것.

기술 스택
- 프레임워크: Next.js (최신 버전, App Router)
- 데이터베이스: Supabase
- UI 컴포넌트: shadcn
- 테마: next-themes (다크 모드 지원)
- 아이콘: Lucide React
- 타입스크립트: 엄격 모드 활성화
